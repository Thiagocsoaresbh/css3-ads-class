# Exercício 3 – Aula 3 (Portfólio Responsivo)

## Objetivo
Evoluir a página da Aula 3 para um portfólio pessoal responsivo e organizado, aplicando HTML semântico, CSS responsivo e interação simples com JavaScript.

A página deve estar visualmente pronta e funcionar corretamente em celular e desktop.

---

## Onde fazer
No mesmo repositório (fork) do aluno, dentro da pasta `aula3`.

---

## O que entregar
Link do repositório do aluno com a pasta `aula3` finalizada (commit e push feitos).

---

## Passo a passo do que fazer

### 1) Personalizar o conteúdo da página (obrigatório)
Editar o `index.html` e fazer as seguintes alterações:

- Trocar o título principal pelo nome do aluno ou nome do portfólio
- Escrever um texto na seção **Sobre mim** com no mínimo 4 linhas
- Alterar os nomes dos 3 projetos e escrever uma descrição para cada um
- Atualizar o rodapé com nome do aluno e ano

---

### 2) Trabalhar com imagem responsiva (obrigatório)
Na seção **Sobre mim**:

- Substituir a imagem atual por uma nova imagem (link ou local)
- Garantir que a imagem não ultrapasse a largura da tela no celular

Dica: pode usar  
`https://picsum.photos/600/300`

---

### 3) Melhorar visual dos projetos (obrigatório)
No `style.css`, realizar melhorias nos cards de projeto:

- Aplicar borda arredondada
- Ajustar espaçamento interno
- Criar um efeito simples ao passar o mouse (hover)

Exemplo do que pode ser feito:
- leve mudança de cor
- pequena sombra
- aumento de escala leve

---

### 4) Responsividade dos projetos (obrigatório)
Garantir o seguinte comportamento:

- No celular: projetos em coluna
- No desktop (a partir de 768px): projetos lado a lado

Isso deve ser feito usando **Flexbox ou Grid**.

---

### 5) Interação do menu no mobile (obrigatório)
No `script.js`:

- O botão **Menu** deve abrir e fechar o menu no celular
- No desktop, o menu deve permanecer visível

---

### 6) Organização visual da página (obrigatório)
A página deve apresentar:

- Espaçamento adequado entre seções
- Conteúdo legível
- Layout sem elementos quebrados
- Navegação funcional

---

### 7) Teste final (obrigatório)
Antes de entregar:

- Abrir `aula3/index.html` no navegador
- Reduzir a tela para simular celular
- Aumentar a tela para simular desktop
- Confirmar que o menu funciona e os projetos se reorganizam

---

## Commit e push
Fazer commit com mensagem:

`Aula 3 - Portfólio responsivo`

Depois fazer push para o GitHub.

---

## Como será a correção

Será verificado:

- Conteúdo personalizado
- Imagem responsiva
- Cards de projeto com melhoria visual
- Responsividade funcionando
- Menu mobile funcional
- Organização geral da página
- Commit realizado no GitHub

---

## Desafio extra (opcional)

- Adicionar mais 3 projetos
- Criar uma nova seção chamada **Habilidades**
- Adicionar link clicável em pelo menos um projeto
